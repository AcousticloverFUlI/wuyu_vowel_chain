# 元音蚁群模拟：内在机制方案

本文档先讨论“内在机制”版本的蚁群模拟，不急着加入地理扩散、小片传播和社会因素。当前目标是把吴语 S0-S3 的主链条和裂化机制转成一个可计算的元音空间模型。

## 1. 当前最重要的演化链条

目前项目里最重要的一条链可以先写成：

`a > ɑ > ɔ > o > u > əu`

在第一版工作拓扑里，因为 `ɷ` 已确认是介于 `o` 和 `u` 之间的后元音，所以可视化和算法节点暂写作：

`a > ɑ > ɔ > o > ɷ > u > əu`

这条链可以拆成两个部分：

1. 单元音链移：`a > ɑ > ɔ > o > ɷ > u`
2. 裂化/双元音化：`u > əu`

前一段是元音在空间中的移动，后一段是从单元音状态进入复合元音状态。因此建模时最好不要把它们混成同一种变化，而是分成两个模块：

- 单元音模拟模块：只模拟 `a/ɑ/ɔ/o/u` 等单元音之间的移动。
- 裂化模拟模块：模拟某些链位、某些声母条件下，单元音状态如何转成 `əu` 及其邻近形式。

这样可以避免把 `ai`、`ɔi` 这类 i 尾复合音误认为这条链上的裂化。

## 2. 与焦磊蚁群模型的关系

焦磊的思路可以概括成三点：

1. 把元音空间看成一个图，每个独立元音是一个节点。
2. 蚂蚁代表语言使用者或音变传播者，在元音节点之间移动。
3. 路径被反复选择后会积累信息素，最后形成较稳定的元音演化路径。

论文中还强调元音空间拓扑结构的重要性：模型结果依赖于元音节点怎样摆放、哪些节点相邻、节点之间距离怎样计算。焦磊使用的是 Schwartz 元音空间拓扑，并用声学距离与感知距离共同决定元音之间的距离。

我们这个项目可以借鉴这个框架，但不必一开始完整复刻 Schwartz 的 33 元音系统。因为我们的目标更具体：解释吴语 S0-S3 中 `a > ɑ > ɔ > o > ɷ > u > əu` 这条后元音链，以及这条链上的裂化。

## 3. 为什么先只做内在机制

“内在机制”指的是只从元音本身的空间位置和音系结构解释变化，不先加入地理传播。

先这样做有三个好处：

1. 可以先检验这条链在元音空间里是否自然。
2. 可以把单元音链移和裂化分清楚。
3. 可以避免模型一开始就被小片、地理距离、方言接触等因素复杂化。

地理和小片因素可以放在第二阶段，用来解释“哪些方言点更容易选择某条路径”，而不是先用来定义路径本身。

## 4. 元音空间拓扑的基本设想

第一版可以建立一个小型拓扑，而不是完整元音图。

根据目前已经确认的标准，第一版拓扑按以下原则处理：

- `ɷ` 是介于 `o` 和 `u` 之间的后元音，放入主链。
- `ɤ/ɯ` 可以作为后不圆唇平行链。
- `ʌ` 可以纳入后元音链。
- `ə` 仍然看作中央化节点，不直接并入后元音主链。
- `əɯ/ɤɯ/ʌɯ/ʌɤ` 算作 `əu` 型裂化的邻近变体。
- `au/ao/aɤ/yo/yɑ/øy/uo/ua/uᴇ` 第一版先不加入讨论。

### 4.1 单元音节点

核心节点：

`a, ɑ, ɔ, o, ɷ, u`

可选扩展节点：

`ɒ, ʊ, ɤ, ɯ, ə, ʌ`

这些节点可以按几个特征定位：

| 特征 | 作用 |
|---|---|
| 高低 | 判断高化、低化 |
| 前后 | 判断前化、后化 |
| 圆唇 | 判断圆唇化、去圆唇化 |
| 央化 | 判断 `ə/ɤ/ʌ` 等中后部音值 |
| 是否边缘元音 | 判断是否沿元音空间边缘移动 |

第一版不一定需要真实 F1/F2/F3 数值，可以先用离散坐标：

| 元音 | height | backness | rounding |
|---|---:|---:|---:|
| `a` | 1 | 1 | 0 |
| `ɑ` | 1 | 3 | 0 |
| `ɔ` | 2 | 3 | 1 |
| `o` | 3 | 3 | 1 |
| `ɷ` | 3.5 | 3 | 1 |
| `u` | 4 | 3 | 1 |
| `ɤ` | 3 | 3 | 0 |
| `ɯ` | 4 | 3 | 0 |
| `ə` | 2.5 | 2 | 0 |
| `ʌ` | 2 | 2.5 | 0 |

这些数值不是最终答案，只是第一版模拟用的工作坐标。后面如果你能补充更精确的标准，我们可以替换。

### 4.2 裂化节点

裂化节点不应和单元音节点完全同类。它们可以看成“单元音核 + 滑动尾”的状态。

当前可以先纳入：

`əu, əɯ, ou, ɤɯ, ʌɯ, ʌɤ`

其中 `əu` 是最核心的目标状态。`əɯ/ɤɯ/ʌɯ/ʌɤ` 先看作 `əu` 型裂化的邻近变体；`ou` 看作 `o` 位上的链上裂化。

不纳入本次裂化核心：

`ai, ɛi, æi, ɔi`

这些是 i 尾复合音，已经在 S0-S3 裂化统计里移出。

暂不纳入本轮讨论：

`au, ao, aɤ, yo, yɑ, øy, uo, ua, uᴇ`

其中 `yo/yɑ/øy/uo/ua/uᴇ` 可能涉及介音、圆唇介音或记录方式问题；`au/ao/aɤ` 先作为非核心复合音另列，不进入第一版后元音链裂化模型。

## 5. 图上的边如何定义

第一版可以只允许几类变化边。

### 5.1 单元音链移边

核心低成本边：

`a -- ɑ -- ɔ -- o -- ɷ -- u`

这些边代表后化、高化、圆唇化的连续过程。

可选边：

`o -- ɤ -- ɯ`

`ɑ -- ɒ`

`ɔ -- ʌ`

`ʌ -- ə`

这些边用来容纳不同方言点的局部变体。其中 `ɤ/ɯ` 是后不圆唇平行链，`ʌ` 纳入后元音链，`ə` 只作为中央化节点。

### 5.2 裂化边

裂化可以看成从单元音节点进入复合音节点。

核心裂化边：

`u -> əu`

可选裂化边：

`o -> ou`

`ɤ -> ɤɯ`

`ə -> əɯ`

`ʌ -> ʌɯ`

`ʌ -> ʌɤ`

这些边不是说历史上一定如此，而是给算法可选路径。算法会根据现代数据支持度、路径成本和信息素积累来决定哪些边更常被选择。

`au/ao/aɤ` 当前不设为第一版裂化边，避免把另一类复合音混进 `əu` 型裂化模型。

## 6. 路径成本

蚂蚁选择路径时不能完全随机。每条边需要一个成本。

第一版路径成本可以由四部分组成：

| 成本项 | 含义 |
|---|---|
| 空间距离成本 | 两个元音在高低、前后、圆唇上的距离 |
| 方向成本 | 是否符合 `a > ɑ > ɔ > o > ɷ > u > əu` 的主方向 |
| 裂化成本 | 从单元音进入复合音需要额外成本 |
| 数据支持奖励 | 如果某条边能解释很多方言点/很多 token，则成本降低 |

也就是说，`ɑ -> ɔ` 成本低，`a -> u` 成本高；`u -> əu` 可以允许，但要作为单独的裂化步骤处理。

## 7. 如何处理“有些字就是与众不同”

你提到有些字的读音可能本来就比较特殊，这一点很重要。模型里可以这样处理：

1. 不强迫每个字都服从同一条路径。
2. 每个字允许有一个“离群成本”。
3. token 很少、只在个别点出现的音值权重降低。
4. 同一方言点、同一链位、同一声母条件下大量出现的音值权重提高。

这样模型就不会因为一两个 odd value 改变整条演化链。

简单说：

> 系统性音变负责解释大多数材料，个别字的特殊读音可以作为低权重偏离项存在。

## 8. 蚁群算法在本项目中的对应关系

| 蚁群算法概念 | 本项目解释 |
|---|---|
| 节点 | 元音状态，例如 `a`、`ɑ`、`ɔ`、`o`、`u`、`əu` |
| 边 | 可能音变，例如高化、后化、圆唇化、裂化 |
| 蚂蚁 | 一次可能的演化尝试，可以对应一个方言点、一个链位或一组字 |
| 信息素 | 某条变化路径被数据支持的程度 |
| 启发式距离 | 元音空间距离和语言学自然度 |
| 蒸发 | 避免少数偶然路径永久支配模型 |
| 最终路径 | 数据中最稳定、解释力最强的演化路线 |

## 9. 第一版模拟建议

第一版不要模拟所有音值，只做两件事：

### 9.1 单元音链移模拟

输入：

- S0-S3 的单元音现代读音。
- 中古链位初始状态。

目标：

- 看现代单元音是否主要落在 `a > ɑ > ɔ > o > u` 这条链上。
- 看哪些方言点已经走到 `u` 附近，哪些还停在 `ɑ/ɔ/o` 附近。

### 9.2 裂化模拟

输入：

- 已经筛选出的清晰裂化值。
- 只保留后元音链相关裂化，排除 `ai/ɛi/æi/ɔi`、`au/ao/aɤ` 和介音待判类。

目标：

- 看哪些路径最常进入 `əu` 或其邻近形式。
- 看 S2/S3 同现是否可以从同一条裂化机制解释。
- 看 N/T 等声母条件是否只是影响裂化概率，而不是决定裂化方向。

## 11. 仍需确认的元音标准

下面这些标准还可以后续继续细化：

1. `ʊ` 是否看作 `u` 的低化/松化变体，还是暂时并入 `ɷ/u` 区间？
2. `ɒ`、`ᴀ`、`ɑ` 与 `a` 的相对位置如何细分？
3. 带 `ʔ`、鼻尾、舌尖元音或音节辅音的值是否全部排除出第一版模型？
4. 如果一个字在某方言点有多个读音，是否一律拆开并降低单个读音权重？
5. 未来是否需要单独建立“圆唇介音化模块”来处理 `yo/yɑ/øy/uo/ua/uᴇ`？

## 12. 当前建议

我建议第一版模型就做一个小范围版本：

1. 只用 S0-S3。
2. 只做单元音链移和后元音链裂化。
3. 主链固定为 `a > ɑ > ɔ > o > ɷ > u > əu`。
4. `ai/ɛi/æi/ɔi` 不作为裂化。
5. `au/ao/aɤ/yo/yɑ/øy/uo/ua/uᴇ` 先另列，不进入第一版核心裂化模型。
6. 方言点和小片暂时只作为结果解释，不作为算法输入。

这样第一版结果会比较干净，能够先回答一个最核心的问题：

> 吴语 S0-S3 的现代分布，能否通过一条以内在元音空间为基础的 `a > ɑ > ɔ > o > ɷ > u > əu` 链式演化路径解释？
