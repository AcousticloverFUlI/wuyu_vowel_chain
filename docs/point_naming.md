# 方言点命名规范与录入注意事项

本文档用于约定每个方言点 CSV 的命名与字段填写规则，保证批量清洗时一致、可追溯。

## 1. 文件与目录

- 目录：`data_raw/points/`
- 文件名建议格式：`{point_id}_{point_name}.csv`
  - 示例：`WZ01_温州.csv`、`RX01_瑞安.csv`
  - 保持稳定，不要随意改名

## 2. point_id 命名规范

- 全局唯一
- 建议 4 位：`[A-Z]{2}[0-9]{2}`（两位字母+两位数字）
  - 示例：`WZ01`、`RX01`
- 如果同一县市有多个调查点，可以用数字区分：`WX01`、`WX02`

## 3. point_name 命名规范

- 中文标准地名
- 与 `point_id` 一一对应
- 避免同名歧义（可补充县级或区级信息）
  - 示例：`无锡`、`无锡-城区`

## 4. subbranch 命名规范

- 采用统一的吴语片系名称
- 统一格式：`片系-小片`（如有小片）
  - 示例：`太湖片-苏沪嘉小片`、`瓯江片`

## 5. 经纬度

- `lat`、`lon` 为十进制度数
- 不知道可留空，但建议后续补齐

## 6. CSV 列结构（每个方言点文件都必须包含）

- `point_id`
- `point_name`
- `subbranch`
- `lat`
- `lon`
- `韵`
- `声组`
- `汉字`
- `读音`
- `note`

## 7. 录入注意事项

- `韵`、`声组`、`汉字`、`读音` 不要留空
- `读音` 为音标字符串，尽量统一格式
- `note` 用于记录空缺项原因或补充信息（如缺项类型、材料来源等）
