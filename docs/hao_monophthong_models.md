# 豪韵单音化与单元音库藏的两个模型

本文解释 `scripts/analyze_hao_models.py` 生成的两个模型结果。输入表是
`data_clean/value_type/point_hao_monophthong_relation.csv`，原始数据不做修改。

## 1. 数据口径

这里分析的是 `rhyme_modern = 豪`。项目映射表中 `豪` 对应 `S5 (*au)`，
不是 `S4`；`S4` 是 `侯 (*əu)`。

斜杠读音已经拆开计算。例如 `u/o` 会拆成 `u` 和 `o` 两个音值，不会被当成
第三种独立音值。当前 `point_monophthong_inventory.csv`、
`point_hao_monophthong_relation.csv`、`hao_value_summary.csv` 以及模型输入表中
没有残留 `/`。

单元音判定口径：

- `ɔ, ɒ, u, ɤ, ɯ, ʌ, ə` 等只含一个元音核的音值算单元音。
- `ao, au, aɤ, ɑɤ, ɑɒ, əw, io` 等算复元音或带滑音形式。
- 带 `ʔ` 的音值不算单元音，因为它们属于入声/塞尾类。

token 数的处理：

- 豪韵单音化模型使用 `hao_monophthong_token_count / hao_total_token_count`。
- 吸引子模型使用豪韵单元音 token 作为观察计数；一个只出现 1 次的 odd value
  对模型影响很小，一个出现几十次的主流读音影响较大。
- `S0-S3` 单元音库藏中每个元音的 token 数保存在
  `point_hao_monophthong_relation.csv` 和
  `hao_attractor_softmax_probabilities.csv` 中，便于人工解释。

## 2. 模型一：豪韵是否单音化

这个模型问的是：一个方言点的豪韵，有多大比例已经读成单元音？

形式上是加权二项 logistic：

```text
豪韵单元音 token 数 ~ Binomial(豪韵总 token 数, p)
```

也就是说，一个方言点如果有 98 个豪韵例字，它对模型的影响会大于只有
1 个零星读音的情况。

模型使用的解释因素是：

- `S0-S3` 是否已经有 `ɔ/ɒ`。
- `S0-S3` 后元音单元音库藏大小。
- `S0-S3` 后元音 token 数。
- `S0-S3` 后元音 token 占全部单元音 token 的比例。

当前系数表在：

`data_clean/value_type/hao_monophthong_logistic_coefficients.csv`

主要结果：

```text
S0-S3 已有 ɔ/ɒ：机会约 ×1.75
后元音库藏大小：机会约 ×1.71
后元音 token 数：机会约 ×2.12
后元音 token 占比：机会约 ×0.34
```

前三项比较直接：如果一个方言点本身已经有较丰富、较高频的后元音系统，尤其有
`ɔ/ɒ` 这类豪韵常见目标，豪韵更容易整体单音化。

最后一项为负，不应解释成“后元音越多越不单音化”。它是在同时控制了后元音库藏大小
和后元音 token 数之后的残余效应。比较合理的语言学理解是：单纯“后元音占比高”还不够，
关键是有没有合适的 `ɔ/ɒ` 目标位置，以及后元音系统本身是否足够丰富。
如果一个系统主要由少数高频 `u/o` 支撑，它未必给 `*au` 提供最自然的单音化落点。

对应图：

`figs/hao_monophthong_logistic_model.png`

读图方法：

- 左图看方向：条形在 0 的右边，表示这个因素会提高豪韵单音化的机会；在左边则表示降低。
- 右图看拟合：横轴是实际观察到的豪韵单音化比例，纵轴是模型预测的单音化概率。
- 点越大，豪韵 token 越多，所以这些点比小点更接近“主流读音”的证据。
- 橙色点表示 `S0-S3` 已经有 `ɔ/ɒ`；绿色点表示没有。
- 越靠近灰色对角线，说明模型预测越接近观察值。

## 3. 模型二：豪韵单音化后落到哪个元音

这个模型问的是：如果豪韵已经有单元音形式，它更可能落到哪个元音核？

它不是直接预测“是否单音化”，而是在候选目标元音之间做选择，例如：

```text
ɔ, ɒ, æ, ʌ, ɐ, ɜ, ə, a, ɯ, u, ɤ
```

模型使用两个因素：

- `S0-S3` 是否已经有同一个元音。
- 该候选元音与 `*au` 单音化方向是否接近。

其中“与 `*au` 接近”用一个粗略音系距离表示：`*au` 的自然单音化目标设在
开中、后、圆唇区域，接近 `ɔ`，并允许 `ɒ` 作为低后圆唇近邻。

当前系数表在：

`data_clean/value_type/hao_attractor_softmax_coefficients.csv`

主要结果：

```text
S0-S3 已有该元音：被选为豪韵目标的机会约 ×4.97
与 *au 单音化方向相近：被选为目标的机会约 ×5.66
```

语言学解释是：豪韵目标不是随机选的。它一方面受历史音值 `*au` 的自然收缩方向限制，
所以强烈偏向 `ɔ/ɒ` 一带；另一方面，如果某个方言点在 `S0-S3` 中已经有同类元音，
该元音就像一个现成的“落点”或“吸引子”。

目标元音的观察与预测汇总在：

`data_clean/value_type/hao_attractor_softmax_summary.csv`

当前最重要的分布是：

```text
ɔ: 观察 3974 token，占 77.86%；模型预测 3529.53 token，占 69.15%
ɒ: 观察 516 token，占 10.11%；模型预测 926.05 token，占 18.14%
```

模型抓住了豪韵主要落在 `ɔ/ɒ` 区域这一点，但仍然会高估 `ɒ` 和少量高后圆唇方向。
这说明只用“库藏吸引”和“音系距离”还不能完全解释 `ɔ` 与 `ɒ` 的地域差异；后续可以加入
小片、地理邻近、或更细的音位特征来解释残差。

对应图：

`figs/hao_attractor_softmax_model.png`

读图方法：

- 左图看“目标元音为什么会被选中”。右侧条形越长，表示这个因素越能吸引豪韵落入该元音。
- `S0-S3 已有该元音` 为正，说明已有库藏会形成吸引子。
- `与 *au 单音化方向相近` 为正，说明音变方向本身也限制目标，主要偏向 `ɔ/ɒ` 一带。
- 右图比较各目标元音的观察 token 数和模型预测 token 数。红色是实际数据，蓝色是模型预测。
- 如果蓝色高于红色，说明模型高估了该目标；如果蓝色低于红色，说明模型低估了该目标。

因此这张图不是地图，也不是每个方言点的分布图；它是在解释“豪韵单音化以后，为什么主要落到
`ɔ/ɒ`，而不是任意一个单元音”。

## 5. 目前最容易讲清楚的结论

可以把结果说成三个层次：

1. 豪韵 `*au` 在多数吴语点中已经单音化，主导结果是 `ɔ`。
2. 是否单音化与该点已有后元音系统有关，尤其与 `S0-S3` 中是否已经有 `ɔ/ɒ` 有关。
3. 地理上，未单音化保留主要集中在毗陵小片；`ɒ` 单音化相对偏向临绍和毗陵，
   `ɔ` 单音化则是更广域的主流结果。

## 6. 给论文表述的简化说法

这两个模型可以分工理解：

- 模型一看“豪韵是否已经从复元音系统脱离，变成单元音”。
- 模型二看“单音化以后，它被哪个已有元音位置吸收，或接近哪个自然音变方向”。

当前结果支持一个比较清楚的解释：豪韵单音化和方言点已有的后元音系统有关，但不是简单的
“后元音越多越单音化”。更准确地说，`S0-S3` 中如果已经存在 `ɔ/ɒ` 这样的后圆唇或低后
单元音位置，豪韵 `*au` 更容易单音化，并且更容易落入这个位置。大量 token 的主流读音
主导模型，少量 odd value 只作为弱信号保留。

## 7. 给纯文科老师的讲法

可以不用先讲模型名，直接讲成两个问题。

第一个问题是：每个地方的“豪”韵，是不是已经从 `au` 这种双元音，变成了一个单元音？

这里我们不是简单数“有几个地方变了”，而是看每个地方的例字数量。比如某地 90 多个豪韵字
几乎都读 `ɔ`，这说明 `ɔ` 是这个地方的主流读法；如果某个读音只出现 1 次，就只当作一个
很弱的信号，不让它左右整体判断。

模型一的结果可以这样说：

```text
一个地方如果本来就有比较成熟的后元音系统，尤其已经有 ɔ 或 ɒ，
那么豪韵更容易单音化。
```

第二个问题是：豪韵单音化以后，为什么多半变成 `ɔ`，有些地方变成 `ɒ`，
而不是随便变成别的元音？

模型二的结果可以这样说：

```text
豪韵的目标元音受两个力量影响：
1. 历史上的 *au 自己比较容易收缩到 ɔ/ɒ 这种后圆唇区域；
2. 如果某个方言点本来已经有这个元音，这个元音就像一个现成的位置，会吸收豪韵。
```

所以这里的“模型”本质上不是神秘算法，而是在检验一个语言学判断：

```text
豪韵单音化 = 历史音变方向 + 现代方言点已有元音系统 的共同结果。
```

地图则补充第三层意思：

```text
这种变化不是随机散布的。未单音化保留明显偏向毗陵小片；
ɔ 是最广泛的主流结果；ɒ 则更集中在临绍和部分毗陵点。
```

## 8. 类型学蕴涵关系

新增脚本：

`scripts/analyze_hao_implications.py`

输出表：

- `data_clean/value_type/hao_implication_point_features.csv`
- `data_clean/value_type/hao_typological_implication_rules.csv`

这里把豪韵先二分：

```text
有单音化成分：monophthong_only + mixed_mono_and_diphthong
完全未单音化：diphthong_or_glide_only
```

目前最值得讲的不是“有 ɔ/ɒ 就必然单音化”。因为 `ɔ` 有反例：

```text
溧阳：S0-S3 有 ɔ，但豪韵为 ɑɤ
吴江芦墟：S0-S3 有 ɔ，但豪韵为 au
```

更稳的蕴涵关系是围绕低后元音 `ɑ/ɒ` 和库藏规模：

```text
S0-S3 有低后元音 ɑ/ɒ  =>  豪韵有单音化成分
17/17，无反例
```

它的逆向必要条件也成立：

```text
豪韵完全未单音化  =>  S0-S3 缺低后元音 ɑ/ɒ
6/6，无反例
```

还可以写成库藏规模蕴涵：

```text
S0-S3 单元音核数 >= 6  =>  豪韵有单音化成分
16/16，无反例

S0-S3 后元音核数 >= 4  =>  豪韵有单音化成分
10/10，无反例

豪韵完全未单音化  =>  S0-S3 单元音核数 <= 5
6/6，无反例

豪韵完全未单音化  =>  S0-S3 后元音核数 <= 3
6/6，无反例
```

地理条件加入以后，还有一个局部蕴涵：

```text
毗陵小片 且 S0-S3 缺低后元音 ɑ/ɒ  =>  豪韵完全未单音化
4/4，无反例
```

这说明“未单音化”不是简单由 S0-S3 库藏单独决定的。因为很多非毗陵点也缺 `ɑ/ɒ`，
但豪韵仍然单音化为 `ɔ`。更准确的说法应该是：

```text
低后元音 ɑ/ɒ 的存在，是豪韵单音化的充分支持条件；
低后元音 ɑ/ɒ 的缺失，是豪韵未单音化的必要条件，但不是充分条件。
在毗陵小片内部，缺 ɑ/ɒ 与未单音化之间的关系最强。
```

换成纯文科表述：

```text
如果一个方言点在 S0-S3 中已经有 ɑ/ɒ 这样的低后元音位置，
豪韵几乎可以确定已经出现单音化。

反过来，凡是豪韵还没有单音化的点，
它们的 S0-S3 都没有 ɑ/ɒ，单元音库藏也比较小。

但“没有 ɑ/ɒ”本身还不能保证豪韵不单音化；
只有在毗陵小片，这个条件才特别强地对应未单音化保留。
```
