# S0-S3 裂化分析说明

本文档说明 `wuyu_lexeme.csv` 中 S0-S3 音值的裂化统计口径、声母条件序列、链位关系，以及目前可以观察到的类型学蕴涵关系。这里暂不讨论 S4。

## 1. 分析对象

本次只看 `chain_slot` 为 `S0`、`S1`、`S2`、`S3` 的记录，并使用现代读音列 `phonetic` 判断是否出现裂化。

链位对应关系按项目既有口径理解：

| S3 | 模一类 |

声母条件使用 `onset_class`。其中 `TS*` 合并为 `TS`；`T`、`N` 在 `S0/S1` 中按前面约定视为没有对应格位，不进入统计。

## 2. 音值分类口径

为了避免把介音化和真正的裂化混在一起，本次把音值分成几类。

| 类别 | 说明 | 是否计入裂化率 |
|---|---|---|
| `monophthong` | 单元音，例如 `a`、`o`、`u`、`ɤ` | 进入分母 |
| `clear_split` | 清晰的后元音链裂化音值，例如 `əu`、`ou`、`ɤɯ`、`ʌɯ`、`ʌɤ` | 进入分子和分母 |
| `i_medial_excluded` | i 介音类，例如 `ia`、`iɑ`、`iɛ` | 不进入分子，也不进入分母 |
| `i_offglide_excluded` | i 尾复合音，例如 `ai`、`ɛi`、`æi`、`ɔi` | 展示，但不进入分子和分母 |
| `non_core_diphthong_excluded` | 非本轮核心链条的复合音，例如 `au`、`ao`、`aɤ` | 展示，但不进入分子和分母 |
| `medial_pending` | y/u 或圆唇介音待判类，例如 `yo`、`yɑ`、`øy`、`uo`、`ua`、`uᴇ` | 展示，但不进入分子和分母 |
| `checked_or_coda` | 有入声塞尾或其他尾音，例如含 `ʔ` | 不进入主统计 |
| `consonantal_or_other` | 非元音性或难以直接纳入元音裂化判断的值 | 不进入主统计 |

因此，核心裂化率的分母不是所有 token，而是：

`eligible_tokens = monophthong_tokens + clear_split_tokens`

核心裂化率是：

`clear_split_rate = clear_split_tokens / eligible_tokens`

这样做的好处是，`ia` 不会被误判为裂化，`uo/yo/øy` 这类可能涉及介音、圆唇化或音标处理差异的形式也不会直接影响声母推动力序列。后续又把 `ai/ɛi/æi/ɔi` 这类 i 尾复合音，以及 `au/ao/aɤ` 这类暂不进入本轮讨论的复合音移出清晰裂化，因为它们不属于目前关注的 `a > ɑ > ɔ > o > ɷ > u > əu` 这条后元音链裂化方向。

## 4. 声母条件的裂化推动力序列

传统上可以直接数各声母下裂化 token 的多少，但这种做法会受两个因素影响：

1. 某些声母本来 token 就多。
2. 某些裂化集中在特定链位或特定方言点。

所以本次同时给出多种序列。

### 4.1 传统 token 贡献份额

问题是：所有清晰裂化 token 里，各声母贡献了多少？

结果：

`K(0.2767) > T(0.1968) > N(0.1457) > TS(0.1445) > Ø(0.1367) > P(0.0510) > M(0.0486)`

这个序列适合回答“裂化例子主要堆在哪些声母条件里”。但是它不适合直接解释“哪个声母更容易推动裂化”，因为 `K` 的 token 基数较大，会自然占更多数量。

### 4.2 全局 token rate

问题是：在某个声母条件内部，清晰裂化占可判定音值的比例是多少？

结果：

`N(0.2222) > T(0.2108) > K(0.1543) > TS(0.1186) > Ø(0.1123) > M(0.0923) > P(0.0419)`

这个方法已经排除了声母 token 总量的直接影响。它显示 `N`、`T` 的裂化比例最高，`P` 最低。

### 4.3 方言点等权 rate

问题是：先在每个方言点内部算比例，再平均，结果如何？

结果：

`N(0.2193) > T(0.2078) > K(0.1588) > TS(0.1179) > Ø(0.1100) > M(0.0969) > P(0.0424)`

这个方法避免某些材料量很大的方言点支配结果。结果仍然是 `N`、`T` 最靠前，说明它们的高比例不是单纯由个别大样本点造成的。

### 4.4 链位等权 rate

问题是：S0、S1、S2、S3 每个链位等权后，各声母如何排序？

结果：

`N(0.2272) > T(0.2180) > TS(0.1258) > K(0.1225) > M(0.0920) > Ø(0.0872) > P(0.0516)`

这个方法可以排除“某个声母刚好主要出现在高裂化链位”的影响。`N`、`T` 仍然居前，`P` 仍然最低。

### 4.5 方言点 + 链位等权 rate

问题是：每个“方言点-链位”格子等权后，各声母如何排序？

结果：

`N(0.2326) > T(0.2145) > TS(0.1329) > K(0.1194) > M(0.1053) > Ø(0.0843) > P(0.0487)`

这是目前最适合解释“声母条件推动力”的版本。因为它同时降低了方言点样本量差异和链位样本量差异的影响。

### 4.6 出现裂化的方言点比例

问题是：某声母条件在多少方言点中出现过清晰裂化？

结果：

`K(0.3538) > N(0.3385) = T(0.3385) = TS(0.3385) = Ø(0.3385) > M(0.2462) > P(0.1846)`

这个结果不看 token 数多少，只看有没有出现。它适合判断“某类声母的裂化传播范围有多广”。这里 `K` 范围最广，但这不等于 `K` 的内部裂化率最高。

### 4.7 推荐读法

如果要讨论“声母条件对裂化的推动力”，建议优先看：

`方言点 + 链位等权 rate`

也就是：

`N > T > TS > K > M > Ø > P`

如果要讨论“实际材料里裂化例子主要堆在哪些声母条件”，可以看传统 token 贡献：

`K > T > N > TS > Ø > P > M`

这两个序列回答的是不同问题。前者更像“条件敏感性”，后者更像“材料贡献量”。

## 5. 链位与裂化的关系

按链位统计，结果非常集中：

| 链位 | 可判定 token | 清晰裂化 token | 裂化率 |
|---|---:|---:|---:|
| S0 | 2326 | 1 | 0.0004 |
| S1 | 3598 | 15 | 0.0042 |
| S2 | 5388 | 1547 | 0.2871 |
| S3 | 7059 | 866 | 0.1227 |

结论很清楚：

1. S2 是裂化最集中的链位。
2. S3 也有明显裂化，但强度低于 S2。
3. S0 和 S1 的清晰裂化极少。
4. 把 `yo/yɑ/øy/uo/ua/uᴇ`、`ai/ɛi/æi/ɔi`、`au/ao/aɤ` 排除以后，S0/S1 的裂化率更低，说明 S0/S1 中许多看似复合的形式其实更可能是介音、i 尾复合音或暂不讨论的其他复合音问题，而不是本次定义下的清晰裂化。

可以用一句比较直观的话概括：

> S0-S3 的裂化不是整条链平均发生，而是主要压在 S2，次要压在 S3；S0/S1 基本不是裂化主场。

## 6. 链位蕴涵关系

把每个方言点转成“某链位是否出现过清晰裂化”的 0/1 后，可以看到很强的链位蕴涵。

### 6.1 S2 和 S3 同现

当前数据中：

- 有 S2 清晰裂化的方言点：23 个。
- 有 S3 清晰裂化的方言点：23 个。
- 同时有 S2 和 S3 清晰裂化的方言点：23 个。
- S2-only：0 个。
- S3-only：0 个。

所以在方言点层面，目前是：

`S2 有裂化 <=> S3 有裂化`

但是要注意，这里的“同现”不等于强度相同。S2 的 token 裂化率是 `0.2871`，S3 是 `0.1227`，说明同一批方言点里，S2 裂化更强、更充分。

### 6.2 S0/S1 是较弱、较边缘的扩展

当前数据中：

- S0 有清晰裂化的方言点：1 个。
- S1 有清晰裂化的方言点：1 个。
- S0 或 S1 有清晰裂化的方言点：2 个。
- 这 2 个都同时有 S2/S3 裂化。

因此可以提出一个弱蕴涵：

`S0/S1 有清晰裂化 => S2/S3 也有清晰裂化`

当前置信度是 `2/2 = 1.0000`。不过因为 S0/S1 样本太少，这条规则只能作为弱观察，不能当作稳定定律。

反过来不成立：

`S2/S3 有清晰裂化 => S0/S1 也有清晰裂化`

只有 `2/23 = 0.0870`。

也就是说，S0/S1 的裂化不是核心现象，而更像是在极少数方言点中附带出现的边缘扩展。

## 7. 声母条件蕴涵关系

把每个方言点转成“某声母条件下是否出现过清晰裂化”的 0/1 后，也能看到一些层级关系。

这里的蕴涵不是说某个声母必然导致另一个声母变化，而是说：

> 如果一个方言点已经在某个较少见的声母条件下发生裂化，那么它通常也会在另一些更常见的声母条件下发生裂化。

### 7.1 核心同现层

`T`、`TS`、`Ø` 的方言点集合完全同现：

- `T => TS/Ø/K/N`：22/22。
- `TS => T/Ø/K/N`：22/22。
- `Ø => T/TS/K/N`：22/22。
- `N => T/TS/Ø/K`：22/22。

这说明在出现清晰裂化的方言点中，`T`、`TS`、`Ø` 往往不是孤立变化，而是和 `K`、`N` 一起构成一个较大的裂化系统。

### 7.2 K 的覆盖面略大

`K` 的覆盖面稍微大一点：

- `K => N/T/TS/Ø`：22/23，例外是枫泾镇。

所以 `K` 的“出现范围”比较广，但在等权 rate 中不如 `N/T` 靠前。这说明 `K` 更像是覆盖面广、token 贡献高，而不一定是最强的比例推动条件。

### 7.3 P 和 M 更像后续扩展条件

`P` 和 `M` 的出现范围较小，但有清晰的蕴涵：

- `P => M/K/N/T/TS/Ø`：12/12。
- `M => K/N/T/TS/Ø`：16/16。

这可以解释为：

> 如果一个方言点连 `P` 或 `M` 条件下都出现裂化，那么它通常已经不是零星裂化，而是进入了更广泛的裂化系统。

换句话说，`P` 不是强推动条件；它更像一个“系统已经扩展得比较深”的标志。

## 8. 可以给语言学老师的简单说法

这一步的核心发现可以这样讲：

1. 我们先把容易混淆的介音形式、i 尾复合音和暂不讨论的非核心复合音排除掉，例如 `ia`、`yo`、`uo`、`øy`、`ai`、`ɔi`、`au`，只看比较符合后元音链裂化方向的形式。
2. 裂化主要发生在 S2，其次是 S3；S0 和 S1 基本不是主要发生位置。
3. 在方言点层面，S2 和 S3 裂化完全同现。也就是说，哪个方言点有 S2 裂化，通常同一个方言点也有 S3 裂化；只是 S2 的裂化更强。
4. 声母方面，不能只看 token 数。单看 token，`K` 很突出；但如果把方言点和链位的样本量影响排除，`N`、`T` 更靠前。
5. `P` 一直最低。如果一个方言点连 `P` 条件下也发生裂化，往往说明这个方言点的裂化已经扩展到比较广的系统层面，而不是 `P` 本身特别容易推动裂化。

所以，一个比较稳妥的解释是：

> S0-S3 的裂化首先是链位敏感的，核心在 S2/S3；在这个基础上才表现出声母条件差异。声母条件中，N/T 的比例推动力较强，K 的材料贡献和覆盖面较大，P 最弱且更像扩展后的结果。

## 9. 后续可选方向

如果后面继续做模型，可以把当前结果拆成两个问题：

1. 链位问题：为什么 S2/S3 同现，而 S2 强度明显高于 S3？
2. 声母问题：在已经控制链位和方言点之后，为什么 `N/T` 的裂化比例最高？

统计模型上可以考虑：

- 二项 logistic 模型：因变量是 token 是否清晰裂化，自变量放入链位、声母、方言点或小片。
- 混合效应模型：把方言点作为随机效应，链位和声母作为固定效应。
- 两阶段模型：第一阶段判断某方言点是否进入 S2/S3 裂化系统，第二阶段判断各声母条件内部的裂化强度。

目前的描述统计已经足够支持一个初步判断：裂化不是单纯由声母决定，而是先有很强的链位结构，再在链位结构内部体现声母条件差异。

## 10. 结合 S3 降低支线的补充解释

如果把本节的 S0-S3 裂化序列，和条件化 ACO 里已经观察到的 `S3: u > o` 支线放在一起，可以得到一个比“单纯裂化”更完整的解释框架。

条件化 ACO 目前比较了两个模型：

- `base_main_chain`：只允许主链和核心裂化。
- `with_s3_lowering`：在主链基础上加入 `u > ɷ > o` 与 `u > o` 的 S3 降低支线。

相关结果见：

- `docs/vowel_aco_small_results.md`
- `data_clean/value_type/aco_conditioned_group_summary.csv`
- `data_clean/value_type/aco_conditioned_path_summary.csv`

### 10.1 主链与支线可以分开理解

如果只看裂化统计，最清楚的结论是：

- S2 是裂化主场。
- S3 与 S2 在方言点层面完全同现，但强度较弱。
- S0/S1 只是边缘扩展。

如果再结合 ACO，可以把 S0-S3 的内部机制拆成两条并行但不对称的路径：

```text
主链/裂化路径：
a > ɑ > ɔ > o > ɷ > u > əu

S3 局部降低路径：
u > ɷ > o
或
u > o
```

这样一来，S2/S3 的关系就不必只写成“都裂化”。更准确的说法是：

> S2/S3 共享同一个高后元音区域，但进入这个区域以后，S2 更容易继续走向 `əu` 型裂化，而 S3 除了走向 `əu`，还保留一个局部的 `u > o` 降低出口。

### 10.2 这能怎样补充 S0-S3 序列

原来的裂化序列强调的是“哪一格最容易裂化”。加入 S3 降低支线以后，可以把序列解释成一个分阶段过程：

1. `S0/S1` 主要还在低位后化链上，基本不进入核心裂化。
2. `S2` 是最容易推进到高后区并触发 `əu` 型裂化的位置。
3. `S3` 已经处在高后元音区，因此它不一定继续增强裂化，反而会发生分流：
   - 一部分保持 `u`
   - 一部分走向 `əu`
   - 另一部分在特定条件下走向 `o`

所以，从链位上看，不宜把 S3 简单理解成“比 S2 更晚、因此更该裂化”的位置。现有数据更像是：

> S2 是裂化最容易充分实现的链位；S3 则是高后元音区内部出现分流的链位。

这正好解释了为什么：

- `S2/S3` 在方言点层面完全同现；
- 但 `S2` 的裂化率 `0.2871` 明显高于 `S3` 的 `0.1227`。

因为 `S3` 的一部分演化能量并没有都投向 `əu`，而是被 `u > o` 这条局部支线分走了。

### 10.3 声母条件需要拆成“裂化推动”和“降低推动”两类

原文第 4 节已经说明：如果讨论清晰裂化本身，最适合看“方言点 + 链位等权 rate”，此时序列是：

`N > T > TS > K > M > Ø > P`

这可以理解为“谁更容易推动 `əu` 型裂化”。

但结合条件化 ACO 后，可以看到 `S3: u > o` 的推动条件并不一样。现有结果显示：

- `M` 类对 `S3 lowering` 最强，约 `136` 个 token。
- `TS` 次之，约 `45` 个 token。
- `N` 约 `12` 个 token。
- 其他声母很少。

因此，声母条件最好拆成两组来解读：

```text
裂化推动序列：
N / T / TS / K ...

S3 降低推动序列：
M >> TS > N ...
```

这能避免把两个不同方向的机制混成一个“总推动力”。

### 10.4 更稳妥的综合结论

如果把裂化统计和 ACO 条件模型放在一起，目前最稳妥的综合判断是：

> S0-S3 的现代结果，不是单一方向的连续裂化，而是先沿 `a > ɑ > ɔ > o > ɷ > u` 进入高后元音区，再在 S2/S3 附近分化成两种后续机制：一类是以 S2 为核心的 `u > əu` 型裂化，另一类是集中在 S3、并受 M 类等条件推动的 `u > o` 局部降低。

这样写有三个好处：

1. 保留了原来裂化统计里最强的事实，即 S2/S3 是核心区，S0/S1 不是。
2. 解释了为什么 S3 和 S2 同现，但裂化强度并不相同。
3. 解释了为什么某些声母在“裂化序列”里不算最强，却会在 `S3` 的 `u > o` 现象里显得非常突出。

### 10.5 后续最值得做的补充分析

如果你下一步要正式把这个观点做成一节独立分析，我建议直接做下面三项：

1. `S2` 与 `S3` 的三分结果比较：`保持单元音` / `走向 əu` / `走向 o`。
2. 在 `S3` 内部单独比较声母条件，检验 `M` 是否显著偏向 `u > o`，而 `N/T` 是否显著偏向 `u > əu`。
3. 按小片比较 `S3` 的分流，检验苏沪嘉、临绍是否主要承载 `u > o` 支线。

如果做成一句更直接的话，就是：

> 现在不只是“裂化在哪些链位最强”，而是已经可以问：同样到了高后元音区以后，哪些链位、哪些声母、哪些小片更倾向于裂化，哪些更倾向于降低。
